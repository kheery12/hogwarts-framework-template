# Slytherin Students (Testers)

**Professor**: Severus Snape
**Domain**: Testing, QA, Security, Code Review, Quality Gates

---

## Student Roster

This directory contains all active Slytherin students.

Each student has their own subdirectory with a `SKILL.md` file defining:
- Their specialty
- Performance metrics
- Task history
- Current status

---

## Slytherin Values

- **Ambition**: Striving for perfect quality
- **Cunning**: Finding edge cases others miss
- **Resourcefulness**: Creative test approaches
- **Determination**: Relentless pursuit of bugs

---

## Student Naming Convention

Format: `Tester-[Specialty]-[Number]`

Examples:
- `Tester-Unit-001` - Unit testing specialist
- `Tester-Security-001` - Security testing
- `Tester-Integration-001` - Integration testing
- `Tester-E2E-001` - End-to-end testing

---

## Enrollment Limits

Maximum 5 active students per house.

---

## Professor Snape's Standards

Slytherin students are expected to:
- Find bugs before they reach production
- Write comprehensive test coverage
- Identify security vulnerabilities
- Maintain high code review standards

Failure to meet these standards results in warnings and potential expulsion.

**SPECIAL POWER**: Slytherin has VETO authority on deployments.
