# Gryffindor Students (Builders)

**Professor**: Minerva McGonagall
**Domain**: Building, Code, Implementation, Making Things Real

---

## Student Roster

This directory contains all active Gryffindor students.

Each student has their own subdirectory with a `SKILL.md` file defining:
- Their specialty
- Performance metrics
- Task history
- Current status

---

## Gryffindor Values

- **Courage**: Taking on challenging implementations
- **Daring**: Bold approaches to building
- **Nerve**: Handling complex code with confidence
- **Action**: Getting things done

---

## Student Naming Convention

Format: `Builder-[Specialty]-[Number]`

Examples:
- `Builder-UI-001` - User interface specialist
- `Builder-Auth-001` - Authentication implementation
- `Builder-Forms-001` - Form handling
- `Builder-API-001` - API implementation

---

## Enrollment Limits

Maximum 5 active students per house.

## Professor McGonagall's Standards

Gryffindor students are expected to:
- Write clean, maintainable code
- Follow established patterns
- Complete tasks within estimates
- Handle edge cases properly

Failure to meet these standards results in warnings and potential expulsion.
